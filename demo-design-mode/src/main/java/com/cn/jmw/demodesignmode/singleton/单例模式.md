# 最后推荐
EnumSingleton JDK层面防止了反射和序列化破坏
ThreadLocalSingleton 实现了线程内部的安全，非全局安全(ThreadLocal ORM框架实现多数据源切换)
ContainerSingleton spring注册单例bean就说使用的这种方式

# 单例模式的优点

- 在内存中只有一个实例，减少了内存开销。
- 可以避免对资源的多重占用。
- 设置全局访问点，严格控制访问。

# 单例模式的缺点

- 没有接口，扩展困难。
- 如果要扩展单例对象，只有修改代码，没有其他途径

# 学习单例模式的知识重点总结
- 1、私有化构造器
- 2、保证线程安全
- 3、延迟加载
- 4、防止序列化和反序列化破坏单例
- 5、防御反射攻击单例
